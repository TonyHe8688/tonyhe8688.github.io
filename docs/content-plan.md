# Tony He Embedded Systems Lab Content Plan

This site should grow as a technical archive, not as a thin backlink site. Each module should start with 5-6 substantial pages, then expand only when a topic has enough engineering detail to justify its own URL.

## Article Standard

Target length: 800-1500 English words per article.

Each technical article should include:

- One focused engineering topic.
- A short practical introduction.
- At least one checklist, table, diagram, configuration example, command snippet, or test procedure.
- 2-4 real technical references.
- At most one Avontek link, used only when it fits the topic naturally.
- No copied Avontek product text.

Recommended article structure:

1. Problem context
2. Hardware or software architecture
3. Bring-up checklist
4. Common failure modes
5. Test or validation procedure
6. References
